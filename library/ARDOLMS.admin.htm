Version Control Systems

Version Control Systems are category of software tools that help a software team manage changes to source code over time.

 Why VCS ?
  1. Collaboration.
  2. Storing versions properly.
  3. Restoring previous version.
  4. Understanding what happened.
  5. Backup

1. Collaboration.

With a VCS everybody on the team is able to work absolutely freely - on any file at any time. The version control system will later allow you to merge all changes in to a common version.

 2. Storing versions properly.

 A VCS acknowledges that there is only one project.

 3. Restoring previous version.

In VCS we can restore previous versions of project that we created.

4. Understanding what happened.

Every time you save a new version of your project, your VCS requires you to provide a short description of what changed. This description helps you to understand what happened in project.

5. Backup

All the time when you stop the project, you save a backup file in your computer, Once your project corrupted or get lots of errors or bugs: you can get your backup project and start project again.


 Version Control System has three major models.
